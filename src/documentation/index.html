What is Dash?

Dash is an open component-based hypermedia system that provides what we believe to be a best of breed set of components. While each component can be used in isolation as less full featured versions of an analogous application, Dash allows users to interoperate components and compose their own workflows without losing data or expending effort while switching between tasks. As hypertext allowed users to flexibly move between texts, Dash allows users to flexibly move between tasks.

Using Dash

Environment

When users first open Dash, they are by default in a collection in freeform perspective (more information on perspectives), which is represented as an unbounded 2D canvas in the middle and toolbars and panels surrounding it.

A. Vertical Toolbar


Description:major components and tools represented as icons. Each icon has an associated flyout panel or popup that show additional details on click

Access: permanently pinned on the left. Flyouts can be toggled open & closed by clicking on the specific icon, or by clicking on the grey drag handle. The popup can be toggled open by clicking on the settings (gear) icon, and closed by clicking the “x” icon on the top right corner.

Objects & actions:

Dashboards - a flyout panel that contains a list of the user’s active dashboards and documents. The user can right-click on the topmost “Dashboard” to invoke the right-click menu, then selecting “Create Dashboard”, in order to create a new dashboard. Each dashboard contains the collections that are within that workspace and the user can choose to expand those collections to view a list of all the items in that collection. The “On-screen Tab” section contains all your currently-opened tabs. Tabs you have closed will be moved to the “Off-screen Tab” section.

Recently Closed - a flyout panel that contains all the documents that the user has recently closed or deleted.
(Since Dashboards and Recently Closed both utilize the tree perspective, documents in these two components can be manipulated in the same way as documents in the tree perspective (more information on tree perspective)

Import- a flyout panel that allows users to import existing files from their local machines, and view files they already have imported. Tools - a flyout panel that contains buttons to create new documents and a color palette to change a document’s background-color (more information on tools) Sharing - (more information on sharing) Filtering - (more information on filters) Presentation Trails - (more information on presentations) Catalog & Help - not yet implemented Settings - a popup that contains user preferences for: Accounts - including resetting the password and linking to Google Modes - including toggling between novice/developer & playground modes and managing groups & permissions Appearances - including setting the background/border/header colors, font & font size for text documents, and header/toolbar appearances (All icons above can be reordered by click and dragging the icon, then dropping it into the desired location in the vertical sidebar) B. Properties Panel Description: panel to set properties and view information of the selected document, divided into collapsible subpanels Access: can be toggled open & close from the context-sensitive toolbar (next section), and resized using the drag handle Objects & actions: An editable document title Options - a subpanel of action buttons to edit various document properties Sharing & Permissions - a subpanel of a table displaying permission and sharing information Fields & Tags - a subpanel of a list of key-value pairs describing the selected document & input box to add custom key-value pairs Context - a subpanel displaying the context information of the document Layout - a subpanel displaying the document layout as a preview More information: (Properties panel) C. Context-sensitive Toolbar Description: toolbar whose contents change based on the type of document selected Access: pinned below the search bar Objects & actions: varies depending on the type of document selected
